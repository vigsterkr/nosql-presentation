NoSQL rendszerek tulajdonságai

  • Flexibilis séma

    • Háromdimenziós leképezés:

    BigTable megjegyzések:

    • kötetlen sor- illetve oszlopszám
    • minden értékhez hozzá van rendelve egy időbélyeg
    • minden értékre egy hármassal hivatkozunk (domain-név, oszlop-név, időbélyeg)

    Dokumentum DB

    • Dokument centrikus tárolás, lekérdezés és adatbázis kezelés
    • Nincs előre meghatározott séma

    MongoDB

    • Collection oriented storage - easy storage of object-style data
    • Full index support, including on inner objects
    • Query profiling
    • Replication and fail-over support
    • Efficient storage of binary data including large objects (e.g. videos)
    • Auto-sharding

    MongoDB felépítése

    • Mongo rendszer = adatbázisok halmaz
    • Adatbázis = kollekciók halmaz
    • Kollekció = dokumentumok halmaza
    • Dokumentum = mező halmaz
    • Mező = kulcs-érték párból áll
    • Kulcs: egyszerű karaktersorozat
    • Érték: string, integer, float, timestamp, binary, dokumentum stb.

    BSON

    • MongoDB JSON document adatbázis
    • Egy Mongo dokumentum ~ sor a relációs adatbázisban, de nincs séma
    • Nincs séma dokumentumok tartalmazhatnak egyedi mezőket.
    • Könnyen importálható bármilyen adabázis, mivel nem kell előfeldolgozni a bemeneti adatot
